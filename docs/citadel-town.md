# Citadel Status AI Town Assets

The `/status` AI Town hero loads local assets from `static/citadel-town/manifest.json`.
Runtime rendering does not depend on remote sprites, iframes, or CDN scripts.

To replace placeholder art:

1. Add new SVG or PNG files under `static/citadel-town/tiles`, `static/citadel-town/buildings`, or `static/citadel-town/avatars`.
2. Update `static/citadel-town/manifest.json` so the existing keys point at the new files.
3. Keep the keys stable unless the renderer or telemetry mapper is updated:
   `ground`, `memory`, `skills`, `automations`, `chat`, `models`, `system`,
   `agentBlue`, `agentGreen`, `agentAmber`, and `agentRose`.
4. Use transparent backgrounds for avatars and buildings so they blend into the dark status panel.

The renderer expects a single ground tile, three to eight small building sprites, and a small avatar set. Larger maps or animated spritesheets should be added behind the manifest loader instead of hardcoding asset URLs in the Svelte component.

Roads are generated by the Pixi renderer from each building's `roadAnchor`, not baked into the ground tile. Keep building positions off the road path and align each `roadAnchor` to the building frontage when changing the town layout.

The town supports clamped drag panning. By default, mobile vertical gestures scroll the page and the town only responds to horizontal dragging. Use the in-canvas move-map toggle to enable full two-axis panning while inspecting a zoomed-in town. Mouse wheel and pinch zoom are disabled; use the in-canvas zoom buttons for zoom in, zoom out, and reset.
